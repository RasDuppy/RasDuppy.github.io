## GAME STATE — AQUA.G (the mutable singleton)

```js
AQUA.G = {
  version:     5,           // Save schema version (bump when state shape changes)
  day:         1,           // Current day number
  fed:         false,       // Was fish fed this day? Reset to false on advanceDay()
  money:       50,          // Player currency ($)
  phase:       'survive',   // Goal phase: 'survive' | 'grow_plant' | 'sell_plant'
  over:        false,       // Game ended (win or loss)
  surf:        45,          // Total biofilm surface area (cm²) from equipment
  tempStable:  false,       // True if heater installed
  anoxicDays:  0,           // Consecutive days with O2 < 1.5 ppm + organic > 0.3

  chem: {                   // Water chemistry (all in ppm unless noted)
    ph:    7.0,             // pH (5–9.5 range, computed from CO2 each tick)
    nh3:   0,               // Ammonia — toxic above 0.2 ppm, lethal ~0.5+
    no2:   0,               // Nitrite — toxic above 0.3 ppm, lethal ~0.8+
    no3:   0,               // Nitrate — safe <40, harmful >80 ppm
    o2:    8.5,             // Dissolved oxygen — min viable ~4, crash <2
    co2:   3.0,             // Carbon dioxide — plants need >5, risk >20
    temp:  76,              // Temperature °F (affects Q10 metabolism scaling)
    org:   0,               // Organic waste load (dimensionless 0–3+)
    bact:  0.05,            // Bacteria colony strength (0–1, starts tiny)
    po4:   0                // Phosphate — algae fuel, harmful >1 ppm
  },

  fish:        [],          // Array of fish instances (see Fish Instance below)
  cleaners:    [],          // Array of cleaner instances (shrimp, snails)
  plants:      [],          // Array of plant instances (live + fake)
  equip:       [],          // Array of equipment keys (strings) owned
  bubbles:     [],          // Visual-only bubble particles for renderer
  algae:       0,           // Algae biomass (dimensionless 0–10+)
  chemHistory: []           // Rolling 60-day array of ChemSnapshot objects
}
```

## AQUA.ITEMS — Species & Equipment Data Table

All items are pure JSON. No functions. The engine reads fields by convention.

### Fish fields
| Field | Type | Meaning |
|-------|------|---------|
| name | string | Display name |
| type | 'fish' | Identifies as fish |
| mass | number | Default mass (g) |
| cost | number | Purchase price ($) |
| o2c | number | O2 consumed per gram per tick (at 76°F) |
| co2p | number | CO2 produced per gram per tick |
| nh3p | number | NH3 produced per gram per tick (×1.8 when fed) |
| wp | number | Organic waste per gram per tick (×1 fed, ×0.3 unfed) |
| po4p | number | PO4 produced per gram per tick |
| phmin/phmax | number | Tolerated pH range |
| nh3l | number | NH3 lethal threshold (ppm) |
| no2l | number | NO2 lethal threshold (ppm) |
| no3l | number | NO3 chronic stress threshold (ppm) |
| o2min | number | Minimum tolerated O2 (ppm) |
| aggression | 0–1 | 0=peaceful, 1=very aggressive |
| schooling | bool | Stressed when fewer than 3 of same species |
| tempMin/Max | °F | Tolerated temperature range |
| behaviors | string[] | Tags: 'livebearing','labyrinth','shrimp','snail','schooling',etc. |
| spriteGroup | string | Which RENDER_ADAPTER draw fn to use |
| desc | string | Store description |
| attrs | string[] | Bullet points (prefix + or - colors them green/red) |

### Cleaner fields
Same as fish but with `type:'cleaner'` and:
| Field | Meaning |
|-------|---------|
| oc | Organic waste consumed per gram per tick |
| o2c | O2 consumed per gram per tick (added v9) |
| co2p | CO2 produced per gram per tick (added v9) |

### Plant fields
| Field | Meaning |
|-------|---------|
| co2c | CO2 consumed per gram per tick at full efficiency |
| o2p | O2 produced per gram per tick at full efficiency |
| no3c | NO3 consumed per gram per tick |
| po4c | PO4 consumed per gram per tick |
| nh3c | NH3 consumed per gram per tick |
| surfPerG | Biofilm surface area added per gram of plant mass |
| lightNeeded | Minimum lightLevel (0–1) to photosynthesize |
| growRate | Mass growth per gram per tick at full efficiency |

### Equipment fields
| Field | Meaning |
|-------|---------|
| onInstall | Array of `{stat, delta}` — applied once on purchase |
| perTick | Array of `{stat, delta}` — applied to calcDeltas delta each tick |
| flags | String array — queried by AQUA.FX.hasFlag(G, flag) |

**Equipment flags in use:**
- `hasFilter` — sponge filter present (bactMult ×1.6 in nitrification)
- `hasHOB` — HOB filter present (same as hasFilter, stronger variant)
- `hasLight` — LED light present (enables full photosynthesis)
- `hasCO2` — CO2 injector present (adds CO2 each tick)
- `hasUV` — UV sterilizer (reduces algae each tick)
- `hasHeater` — heater present (sets tempStable flag)

### Consumable fields
| Field | Meaning |
|-------|---------|
| onUse | Array of action descriptors, processed by AQUA.FX.applyUse() |

**onUse actions:**
- `{action:'boostPlants', delta}` — adds delta mass to all live plants
- `{action:'addChem', stat, delta}` — adds delta to G.chem[stat]; triggers shrimp safety warnings
- `{action:'algaeDose', factor}` — multiplies algae by factor; stresses shrimp
- `{action:'reduceStat', stat, factor}` — multiplies G.chem[stat] by factor; kills shrimp if aquarium salt
- `{action:'boostFishHealth', delta}` — adds delta health to all alive fish

## AQUA.SIM — Pure Simulation Engine

**Contract:** `tick(G, fed)` returns `{newState, events[], delta}`. Never mutates G directly. No DOM access.

### calcDeltas(chem, fed, G) → delta object

Computes per-tick rates. Returns:
```js
{
  o2, co2, nh3, no2, no3, po4, org, bact, algae, anoxic,
  totalSurf  // passed through for debug panel
}
```

**Computation order inside calcDeltas:**
1. Fish respiration & waste (Q10 temperature scaling: `Math.pow(2,(temp-76)/10)`)
2. Nitrification: Michaelis-Menten NH3→NO2 (Nitrosomonas), NO2→NO3 (Nitrobacter)
3. Bacteria colony growth/death:
   - Growth: `0.004 × wasteFuel × o2Factor × surfCap`
   - Death: `0.015 × (1 - o2/2)` when o2 < 2
4. Bacteria O2 consumption: `bact × 0.08`
5. Anaerobic flag: if o2 < 1.5 AND org > 0.3 → `d.anoxic = 1`
6. Decomposition: org × 0.15 + cleaner oc values; cleaners consume O2 + produce CO2
7. Plant photosynthesis: `eff = lightLevel × min(1, co2/5) × min(1, no3/15)` — Liebig minimum
8. Equipment perTick (bubbler, HOB, UV, CO2 injector)
9. Algae growth: `0.09 × min(1,no3/25) × min(1,po4/1.2) × lightFactor`
10. Surface gas exchange: `(9 - o2) × 0.05`

### redoxScore(G) → 0–1
```
(o2/9)×0.6 + (1 - min(1, org/2))×0.3 + (1 - min(1, nh3/0.5))×0.1
```

### tick(G, fed) — events emitted
| Event type | Trigger |
|------------|---------|
| `chem_warn` | nh3>0.2, no2>0.3, po4>0.7, o2<4 — includes `reason` string |
| `chem_info` | no3>20 |
| `algae_bloom` | algae > 1.8 |
| `algae_stress` | algae > 4 (stresses fish) |
| `bact_warn` | bact < 0.08 after day 3 |
| `anoxic_event` | anoxicDays >= 2 (severe if >= 4) |
| `fish_died` | fish.health reaches 0 |
| `phase_change` | day>=30 → 'grow_plant'; javafern.mass>=4.5 → 'sell_plant' |
| `game_over` | all fish dead (won:false) or plant sold at sell_plant phase (won:true) |

### Fish health calculation (per tick)
```
stressIndex = 0
  + if nh3 > nh3l×0.6:  (nh3/nh3l) × 20
  + if no2 > no2l×0.6:  (no2/no2l) × 15
  + if o2  < o2min×1.2: ((o2min-o2)/o2min) × 30
  + if pH out of range:  +10
  + if schooling & < 3 of same species: (3 - count) × 4

stress = clamp(stress + stressIndex - 5, 0, 100)
health -= stress × 0.02
```

## AQUA.shop

```js
AQUA.shop.open()            // Show store div, call build()
AQUA.shop.close()           // Hide store, call advanceDay() (closing costs a day)
AQUA.shop.build()           // Refresh store display and sell section
AQUA.shop.showCat(cat)      // Switch category: 'fish'|'cleaner'|'plant'|'equip'|'other'
```

**Category mapping (`_catFor`):**
- `fish` → fish tab
- `cleaner` → cleaner tab
- `plant` → plant tab
- `equip` → equipment tab
- `consumable`, `substrate`, `decor` → other tab

**Purchase flow:**
1. Deduct cost from G.money
2. If equip: push key to G.equip, call FX.applyInstall
3. If fish: push makeFish() to G.fish
4. If cleaner: push makeCleaner() to G.cleaners
5. If plant: push plant object to G.plants (type:'plant', key, mass from ITEMS, x random)
6. If consumable: call FX.applyUse(item.onUse, G)
7. emit status_changed

## KNOWN ISSUES / GAPS (as of v9)

1. **`cleaners[].health` not initialized in `makeCleaner()`** — v9 FX.applyUse writes it but it starts undefined. Fix: add `health:100` to makeCleaner.
2. **Bacteria bact > 1 not possible** — clamped in tick but growthRate could theoretically push above 1. Currently fine.
3. **pH model oversimplified** — no KH/GH buffering. pH computed purely from CO2 via `8.3 - log10(co2×0.5)×0.8`. Carbonate hardness (KH) not yet modeled.
4. **Temperature is static** — G.chem.temp is always 76°F; heater only sets tempStable flag. No actual temp dynamics yet.
5. **Day/night cycle absent** — photosynthesis runs all day; plants don't consume O2 at night. This is a known gap.
6. **Algae delta warn event fires every tick** if algae>1.8 — should throttle or gate to once per N days.
7. **Dead fish stay in G.fish array** — never cleaned out. They still occupy `alive:false` slots forever. Need a cleanup or graveyard.
8. **Schooling stress uses same-key count** — doesn't account for interspecies schooling (e.g., Corydoras may school with other Corydoras variants).
